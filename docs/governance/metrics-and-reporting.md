# Metrics and Reporting

## Purpose
Define metrics used to monitor system development and performance.

## Types of Metrics

- Requirements coverage
- Verification progress
- Defect density
- Risk exposure
- Test success rate

## Example Metrics

| Metric | Description |
|---|---|
| % Requirements Verified | Coverage of verified requirements |
| Open Risks | Number of active risks |
| Test Pass Rate | Successful test percentage |

## Reporting

Reports should be:
- periodic
- structured
- traceable

## Engineering Implications

- enables monitoring
- supports decision making
- improves transparency
